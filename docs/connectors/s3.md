# AWS S3 Connector

The S3 connector uses [aiobotocore](https://github.com/aio-libs/aiobotocore) under the hood.

## Installation

```bash
uv add fennflow
```

or

```bash
pip install fennflow
```

## Configuration

```python
from fennflow.connectors import S3ConnectorConfig

config = S3ConnectorConfig(
    aws_access_key_id="your-access-key",
    aws_secret_access_key="your-secret-key",
    endpoint_url="https://s3.amazonaws.com",  # optional, for S3-compatible storage like MinIO
    )
```

Credentials can also be provided via the AWS credential chain — environment variables,
`~/.aws/credentials`, IAM roles, etc. See the
[AWS documentation](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-envvars.html)
for the full list of supported options. In that case, just use:

```python
from fennflow.connectors import S3ConnectorConfig

config = S3ConnectorConfig()
```

## Quick Start

```python
import asyncio

from fennflow import ConfigDict, UnitOfWork
from fennflow.backends import InMemoryBackendConfig
from fennflow.connectors import S3ConnectorConfig
from fennflow.files import TextContent
from fennflow.repositories import DeleteRepository, GetRepository, PutRepository
from fennflow.repositories.fields.s3 import S3RepoField


class UserFiles(
    PutRepository,
    DeleteRepository,
    GetRepository,
    ):
    pass


class UOW(UnitOfWork):
    user_files = S3RepoField(UserFiles, bucket_name="my-bucket")
    config = ConfigDict(
        backend=InMemoryBackendConfig(),
        connector=S3ConnectorConfig(),  # uses AWS credential chain
        )


async def main():
    async with UOW() as uow:
        file = TextContent.from_content("Hello, World!")

        await uow.user_files.at("user1/").put(file)

        response = await uow.user_files.at("user1/").get(file.filename)
        file = response[0]
        print(file.content)  # "Hello, World!"

        await uow.user_files.at("user1/").delete(file.filename)


if __name__ == "__main__":
    asyncio.run(main())
```

(This example is complete, it can be run “as is”, assuming you’ve installed the fennflow package)

## Notes

- `endpoint_url` can point to any S3-compatible storage — MinIO, LocalStack, Yandex Object Storage, etc.
- All operations are async via aiobotocore
- Extra kwargs passed to repository methods are forwarded directly to the underlying aiobotocore call
